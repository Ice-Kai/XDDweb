## ⭐ 最新关键认知（2026-06-26，必读，踩了十几轮的坑）
1. **🔴 Astro `<style>` 作用域 vs JS 动态元素**：后台 `admin/[...path].astro` 大量用 `document.createElement` 动态建表/列表。Astro 默认给 `<style>` 加 `[data-astro-cid]` 作用域，**只匹配静态 HTML**，动态元素没这属性 → CSS 类**完全不生效**（表现：灰框默认按钮、封面巨图、列不对齐）。**已把该页 `<style>` 改成 `<style is:global>`**。改后台动态元素样式：要么靠 is:global，要么直接内联 `el.style.cssText`（最稳，批量表格就是全内联）。
2. **🔴 Codex 并行做前端**：另一个 agent(Codex) 同时在改前台。**4321 端口是 Codex 的 `npm run dev`**（前端 HMR）。dev 模式对这个 4000 行巨型后台文件样式注入不可靠 → **验证后台必须用生产构建**：`PORT=4322 node dist/server/entry.mjs` → 看 http://127.0.0.1:4322/admin（无 `data-vite-dev-id` 才是生产）。**别杀 4321 的 node（那是 Codex）；别开自己的 dev**。
3. **分工边界**：我=`admin/[...path].astro`+`api/admin/*`+`api/download`+DB/迁移；Codex=`Base.astro`/`layouts`/`c/*`/`index`/`member/*`/`global.css`。共用 `lib/{content,member,db,ratelimit,auth,security}.ts`（Codex 已重构，导出名变了：`db`/`md5`/`rateLimit`/`createMemberToken`/`verifyMemberToken`，改前先看）。⚠️ 我动过 `c/model.astro`(前台分类同步)，是 Codex 地盘，可能冲突待协调。

## 9. 坑 / 注意
- 构建慢(~3-4min)；改前端用 dev+curl 验证，别每次 build。
- `curl` 测中文要 `encodeURIComponent`，否则中文不编码会"搜不到"(曾误判)。
- dev 模式 HTML 有 `data-astro-source-*` 注入，grep `class="x">文字` 会失配，改 grep 纯文本或 label。
- 改任何鉴权：**别迁回 xuedda.members**；新写密码保持旧站 md5 格式。
- `legacy` 库是真实数据，**测试插入务必带可识别前缀并删除**（之前用 `zz_test_` 验证后已清）。
- 数据安全是用户红线：动 `legacy.*` 前三思，删/改前先 SELECT 确认。
